# AGENTS.md

## 项目概览

这是一个面向个人艺术家品牌的 AI 数字漫剧作品集网站。

网站的核心不是传统求职简历或销售页面，而是通过电影式视听语言，呈现一个持续生长的个人视觉宇宙。内容重点依次为：

1. 个人创作宣言与代表性视觉风格；
2. 视觉宇宙与风格实验；
3. 原创 AI 漫剧系列；
4. 角色与世界观档案；
5. 商业漫剧作品；
6. 创作过程、个人介绍与联系方式。

已确定的首屏创作宣言：

> 在万千位面，寻找一个从未写下的故事。

英文副文案：

> Across countless realms, searching for a story never written.

艺术家显示名称尚未确定。未经用户明确提供，不得擅自虚构姓名、艺名、客户、奖项、经历或联系方式。

## 内容与页面分离

作品、角色和站点文字应由数据文件集中管理，不要把大量作品内容硬编码进 `index.html`。

默认数据文件：

- `data/site.js`
- `data/projects.js`
- `data/characters.js`

为保证用户可以直接双击预览，并避免本地 `fetch()` 限制，第一版使用普通脚本挂载全局只读数据，例如：

```js
window.PORTFOLIO_PROJECTS = [
  {
    id: "example-project",
    type: "original-series",
    title: "示例作品",
    year: "2026",
    status: "正在创作",
    cover: "assets/projects/example-project/cover.webp",
    description: "明确标注为示例的说明。",
    gallery: []
  }
];
```

要求：

- 数据对象字段保持一致；
- 渲染代码不得修改原始数据；
- 缺少图片、视频或字段时优雅降级；
- 不得因单个作品数据错误导致整页无法显示；
- 示例内容必须明确标注为“示例”或“待替换”；
- 不得捏造真实项目描述。

作品类型使用以下固定值：

```text
visual-experiment
original-series
character-archive
commercial-project
behind-the-scenes
```

## 视觉方向

关键词：

- 神秘；
- 电影感；
- 跨位面叙事；
- 诗意；
- 克制；
- 精致；
- 有留白；
- 具有连续世界观。

避免：

- 廉价赛博霓虹；
- 过量渐变；
- 高频故障效果；
- 大量粒子覆盖内容；
- 每个元素都弹跳或发光；
- 通用模板式卡片；
- 视觉效果压过作品本身。

页面色彩节奏：

- 首屏使用深黑、炭灰、暗银、冷白与单一低饱和强调色；
- 向下滚动后逐渐转为更简洁、明亮、留白充足的作品区域；
- 深浅色过渡应自然，不得突然闪切；
- 强调色数量保持克制。

字体规则：

- 中文主宣言可以非常大；
- 英文翻译应小、细、字距较大；
- 正文优先使用系统字体栈；
- 未经用户授权不得引入商业字体；
- 最多保持两类主要字体角色：标题与正文；
- 确保中文在 Windows、macOS、Android 和 iOS 上均可阅读。

## 首屏视听设计

首屏应像 10 至 20 秒的电影序章，而不是普通轮播图。

建议节奏：

1. 黑场或极暗画面；
2. 用户点击“进入故事”；
3. 环境声或极轻钢琴淡入；
4. 角色、场景或分镜碎片缓慢浮现；
5. 中文宣言显现；
6. 英文副文案淡入；
7. 用户滚动后，画面与声音自然过渡到作品区。

视觉素材可包括：

- 代表角色特写；
- 场景切片；
- 眼神、手部、服装等细节；
- 分镜碎片；
- 角色剪影；
- 轻微胶片颗粒；
- 缓慢推近；
- 遮罩、溶解或景深变化。

禁止：

- 强制等待完整开场播放完毕；
- 阻止用户滚动；
- 高频闪烁；
- 快速连续切图；
- 首屏同时加载多段高清视频；
- 在没有用户操作时播放有声媒体。

在真实素材尚未提供时：

- 使用轻量、清楚标注的占位层；
- 不从网络随机下载所谓“AI 艺术图”；
- 保持替换接口简单；
- 在 `README.md` 中说明素材替换位置。

## 音频规范

音频服务于氛围，不是核心内容。

基础实现优先使用：

- HTML `<audio>`；
- JavaScript `play()`、`pause()` 和 `volume`；
- `localStorage`；
- `visibilitychange`。

只有当普通 `<audio>` 无法自然实现多音轨混合、精确淡入淡出或滤波时，才使用浏览器内置且免费的 Web Audio API。

必须满足：

- 首次进入默认无声；
- 只有用户点击或触摸“进入故事”后才能播放；
- 页面始终提供明确的声音开关；
- 声音按钮有文字或可访问名称，不能只靠抽象图标；
- 记住用户的静音偏好；
- 页面进入后台时暂停或降低背景音乐；
- 播放作品视频时暂停或降低背景音乐；
- 视频结束或关闭后平滑恢复；
- 音频加载失败不影响页面内容；
- 背景音量默认非常低；
- 不循环播放明显突兀的短音效；
- 不为每次悬停都播放声音；
- 不使用未确认授权的音乐。

## 作品展示与放大

视觉实验区使用有节奏的错位网格：

- 大图和小图穿插；
- 横图和竖图混排；

必须支持：

- 上一张和下一张；

- 打开时锁定背景滚动；
- 关闭后将焦点返回触发元素；

- 图片加载失败提示；

重点原创漫剧可使用独立静态详情页：

```text
projects/<project-id>/index.html
```

普通视觉实验不必为每张图建立页面。

## 视频规范

- 使用 `<video>` 原生元素；
- 默认 `preload="metadata"`；
- 提供 `poster`；
- 移动端使用 `playsinline`；
- 有声音的视频不得自动播放；
- 首屏背景视频必须静音；
- 不同时自动播放多个视频；
- 滚动离开后可暂停非关键视频；
- 视频必须有标题、简介，重要对白应提供字幕或文本说明；
- 不把高清视频全部打包进首次加载；
- 无视频时应显示封面和说明，而不是空白区域。

推荐格式：

- MP4 / H.264 作为主要兼容格式；
- WebM 仅作为可选补充；
- 图片优先 WebP，必要时提供 JPEG/PNG 后备。

## 动画与滚动

动画应服务于叙事。

优先使用：

- opacity；
- transform；
- clip-path 或简单遮罩；
- CSS transition；
- CSS animation；
- `IntersectionObserver`；
- 必要时使用 `requestAnimationFrame`。

禁止：

- 在 `scroll` 回调中持续进行大量布局计算；
- 使用动画永久占用高 CPU；
- 强制整站横向滚动；
- 通过 JavaScript 模拟浏览器原生平滑滚动；
- 为装饰效果引入大型动画库；
- 动画结束前隐藏核心内容。

时间建议：

- 普通界面反馈：180–400ms；
- 章节过渡：600–1200ms；
- 首屏镜头：4–12s。

必须实现：

```css
@media (prefers-reduced-motion: reduce)
```

减少动态模式下：

- 关闭视差；
- 关闭自动镜头运动；
- 取消非必要循环动画；
- 缩短过渡；
- 保证所有内容立即可读。

- 不出现水平页面溢出；

至少检查以下宽度：

- 360px；
- 768px；
- 1024px；
- 1440px。

## 性能要求

首屏先显示文字与静态封面，再延迟加载大型媒体。

必须：

- 图片设置明确宽高或 `aspect-ratio`，减少布局跳动；
- 非首屏图片使用 `loading="lazy"`；
- 视频使用 `preload="metadata"` 或 `none`；
- 不一次性下载所有正片；
- 不加载未使用的字体、脚本和样式；
- 避免重复素材；
- 压缩图片、视频和音频；
- 对资源加载失败提供降级；

建议资源目标：

- 首屏关键资源尽量不超过约 2–3 MB；
- 缩略图通常控制在 600–900px 宽；
- 列表大图通常控制在 1200–1600px 宽；
- 全屏图通常不超过约 2400px 宽；
- 不直接提交相机或生成工具导出的超大原始文件。

发现明显超大媒体时，不要静默提交；应报告文件大小并建议压缩。

## HTML、CSS 与 JavaScript 代码规范

### HTML

- 使用语义标签：`header`、`nav`、`main`、`section`、`article`、`footer`；
- 每页只有一个主要 `h1`；
- 不使用内联事件处理器，如 `onclick=""`；
- 尽量不写内联样式；
- 外部链接在新窗口打开时添加合适的 `rel`；
- 页面语言设置为 `zh-CN`；
- 声明 UTF-8；
- 保持 DOM 层级简单。

### CSS

- 使用 CSS 自定义属性管理颜色、字号、间距、层级和动画时间；
- 类名使用可读的 `kebab-case`；
- 避免 `!important`，除非有明确理由；
- 避免深层选择器和依赖 DOM 顺序的脆弱样式；
- 不使用无解释的魔法数字；
- 同一视觉规则只维护一份；
- 动画优先改变 `transform` 和 `opacity`；
- 在 `responsive.css` 中集中处理主要断点。

### JavaScript

- 使用现代但广泛支持的原生 JavaScript；
- 默认使用 `const`，确需重新赋值时使用 `let`；
- 禁止使用隐式全局变量；
- 函数保持单一职责；
- DOM 查询集中并检查元素是否存在；
- 事件监听器避免重复绑定；
- 错误应被局部隔离并提供降级；
- 不使用 `eval()` 或动态执行字符串代码；
- 不把大型 HTML 字符串无保护地插入不可信内容；
- 若内容未来来自用户输入，使用 `textContent` 而不是 `innerHTML`；
- 第一版避免 ES modules，以支持直接本地预览；
- 不使用 `fetch()` 读取本地 JSON 作为核心路径。

这是静态展示网站，不应收集敏感数据。

- 第一版联系入口使用普通邮箱或社交链接；
- 不添加表单后端；
- 不提交密码、令牌、API 密钥或私人账号信息；

## 本地预览

网站应尽量支持直接双击 `index.html` 进行基础预览。

完整检查推荐在项目根目录运行一个免费本地静态服务器：

```bat
py -m http.server 8000
```

若 `py` 不可用，可尝试：

```bat
python -m http.server 8000
```

然后访问：

```text
http://localhost:8000
```

不得为了本地预览要求安装 Node.js 或付费软件。

## 修改流程

每次工作遵循：

1. 阅读本文件和相关代码；
2. 简述准备修改的文件与原因；
3. 进行最小范围修改；
4. 保留既有设计语言；
5. 检查控制台错误；
6. 检查键盘操作；
7. 检查移动端；
8. 检查声音开关和视频互斥；
9. 检查减少动态模式；
10. 汇报修改内容、验证结果和仍需用户提供的素材。

不要在同一任务中顺便重构无关代码。

## 验证清单

完成任何界面或功能修改后，至少验证：

### 基础

- `index.html` 可打开；
- 浏览器控制台无明显错误；
- 所有本地路径有效；
- 页面没有水平溢出；
- 缺少媒体时不会整页崩溃。

### 首屏

- 创作宣言正确；
- 未经交互不会播放有声媒体；
- “进入故事”可由鼠标、键盘和触摸触发；
- 声音开关可随时使用；
- 用户可立即向下浏览。

### 作品

- 作品由数据文件渲染；
- 空列表有友好提示；
- 图片可打开和关闭；
- Esc 与焦点恢复正常；
- 视频播放时背景音乐正确降低或暂停。

### 响应式与无障碍

- 360px、768px、1024px、1440px 下主要内容可用；
- Tab 顺序合理；
- 焦点清晰；
- `prefers-reduced-motion` 生效；
- 图片替代文本合理；
- 控件具有可访问名称。

### 性能

- 非首屏图片懒加载；
- 视频没有全部预载；
- 没有新增大型依赖；
- 没有无意提交超大原始素材。

若无法执行某项验证，必须明确说明，而不是声称已经通过。

- 输出目录为仓库根目录；
- Cloudflare Pages 可直接发布静态文件；

## 禁止事项总结

不得：

- 引入框架或构建工具；
- 引入当前或未来可能收费的服务而不先询问；
- 使用无授权素材；
- 自动播放有声媒体；
- 虚构艺术家资料、客户和作品；
- 把原创内容降级为商业服务陪衬；
- 牺牲移动端、键盘操作或减少动态支持；
- 用复杂代码实现浏览器已有的简单能力；
- 为视觉效果造成明显卡顿；
- 修改无关文件；
- 隐瞒未验证事项；
- 使用含糊占位内容冒充最终文案。
